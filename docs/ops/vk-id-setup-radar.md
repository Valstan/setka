# Регистрация VK ID-приложения «Вход вМалмыже» (Радар-ID) — для владельца

> Разовая настройка ~15 минут, по образцу сабантуевской (`SabantuyMalmyzh/docs/ops/vk-id-setup.md`,
> там всё получилось — App ID 54656174 работает). Это **новое отдельное** приложение, потому что:
> (1) VK ID-приложение привязано к базовому домену — у Сабантуя это `сабантуймалмыж.рф`, а Радару
> нужен `вход.вмалмыже.рф`; (2) на экране входа пользователь видит имя приложения — для SSO всей
> экосистемы оно должно быть нейтральным, а не «Сабантуй Малмыж»; (3) не трогаем работающий вход
> Сабантуя. **Зоопарка не будет:** это приложение — единственное на всю экосистему (Вариант А,
> ADR-0002 §4); когда Сабантуй перейдёт на вход через Радар-ID (Ф2/Ф3), его приложение выведем.

## Шаги

### 1. Создать приложение

- https://id.vk.com/about/business/go → войти → **Создать приложение**.
- Тип/платформа: **«Веб»** (не Mini App, не мобильное).
- Название: **`Вход вМалмыже`** (его увидят люди на экране «Войти через VK»).

### 2. Домен и redirect (punycode, символ-в-символ!)

- **Базовый домен:**
  ```
  xn--b1ae3a1a.xn--80adkdyec4j.xn--p1ai
  ```
  (это `вход.вмалмыже.рф` в техническом виде).
- **Доверенный redirect URL:**
  ```
  https://xn--b1ae3a1a.xn--80adkdyec4j.xn--p1ai/auth/vk/callback
  ```

### 3. Передать мне

| Значение | Куда |
|---|---|
| **ID приложения** (число) | можно в чат — публичный |
| «Защищённый ключ» | **никуда не слать** — для VK ID PKCE-обмена secret не нужен (R16); просто сохраните у себя |

### 4. Что сделаю я

Пропишу на проде в `/etc/setka/setka.env`:

```
RADAR_ID_VK_APP_ID=<ID приложения>
```

и кнопка «Войти через ВКонтакте» на логин-странице Радара оживёт
(`/auth/vk/login` → id.vk.ru → `/auth/vk/callback` → сессия).

## Примечание

Приложение заработает только когда домен `вход.вмалмыже.рф` укажет на хост setka
и на нём будет TLS (VK редиректит только на https). Это часть деплоя Ф1 —
см. PENDING «Радар-ID → Ф1 деплой».
