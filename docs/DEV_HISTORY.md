6. **Регулярная ревизия** — каждые 10-15 записей проводить очистку устаревшего

### [2026-04-08] — Адаптация документации из Postopus + внедрение тестирования

**Что изменено:**
- **ДОКУМЕНТАЦИЯ: Создан `docs/AI_DEV_GUIDE.md`** — полное руководство для AI-разработчиков, адаптированное из `postopus/startai.md` для архитектуры SETKA:
  - Структура проекта SETKA (FastAPI + Celery + PostgreSQL вместо MongoDB)
  - Правила безопасности (секреты только в env vars через `config/runtime.py`)
  - AI self-review чек-лист
  - Процедура синхронизации с git в начале сессии
  - Технологический стек SETKA
  - Карта документации

- **ДОКУМЕНТАЦИЯ: Создан `docs/DEV_HISTORY.md`** (этот файл) — журнал изменений проекта по образцу `postopus/DEV_HISTORY.md`:
  - Формат записей с датой, описанием, затронутыми файлами
  - Обратная совместимость
  - Влияние на другие модули

- **ДОКУМЕНТАЦИЯ: Создан `docs/TESTING.md`** — полное руководство по тестированию SETKA:
  - Разделение unit-тестов и интеграционных тестов
  - Инструкция по написанию тестов с моками
  - Тестовый полигон (VK_TEST_GROUP_ID)
  - CI/CD pipeline через GitHub Actions

- **ТЕСТИРОВАНИЕ: Создана инфраструктура тестирования:**
  - `pytest.ini` — конфигурация pytest
  - `tests/conftest.py` — фикстуры и моки для БД, VK API, Redis
  - `.pre-commit-config.yaml` — хуки black, isort, flake8
  - `.github/workflows/ci.yml` — CI/CD pipeline

- **ТЕСТИРОВАНИЕ: Созданы начальные unit-тесты:**
  - `tests/test_config/test_runtime.py` — тесты config/runtime.py
  - `tests/test_filters/test_base_filter.py` — тесты базового фильтра
  - `tests/test_deduplication/test_detector.py` — тесты дедупликации

- **ДОКУМЕНТАЦИЯ: Обновлены существующие документы:**
  - `docs/README.md` — добавлены ссылки на новые документы
  - `docs/START_HERE.md` — улучшена процедура начала сессии разработки

**Затронутые файлы:**
- `docs/AI_DEV_GUIDE.md` — новый
- `docs/DEV_HISTORY.md` — новый (этот файл)
- `docs/TESTING.md` — новый
- `docs/README.md` — обновлён
- `docs/START_HERE.md` — обновлён
- `pytest.ini` — новый
- `tests/conftest.py` — новый
- `tests/test_config/test_runtime.py` — новый
- `tests/test_filters/test_base_filter.py` — новый
- `tests/test_deduplication/test_detector.py` — новый
- `.pre-commit-config.yaml` — новый
- `.github/workflows/ci.yml` — новый

**Причина изменения:**
- **Проблема #1:** SETKA не имела формальной инфраструктуры тестирования — все `scripts/test_*.py` были ручными интеграционными скриптами, требующими живую БД и API
- **Проблема #2:** Не было CI/CD pipeline — код попадал в main без автоматической проверки
- **Проблема #3:** Не было pre-commit хуков — код мог не соответствовать стандартам форматирования
- **Проблема #4:** Документация SETKA не содержала AI self-review чек-листа, правил ведения истории разработки и структурированного руководства для AI-сессий
- **Решение:** Адаптированы лучшие практики из Postopus (old_postopus) с учётом архитектуры SETKA (FastAPI/Celery/PostgreSQL вместо MongoDB)

**Влияние на другие модули:**
- Все новые модули должны покрываться тестами
- CI/CD будет автоматически запускать тесты при push
- Pre-commit хуки обеспечат единый стиль кода
- AI_DEV_GUIDE.md станет отправной точкой для каждой AI-сессии

**Обратная совместимость:**
- ✅ **ДА** — все изменения аддитивны, существующий код не затронут
- `scripts/test_*.py` остаются как были (ручные интеграционные скрипты)
- Новые `tests/` — параллельная инфраструктура unit-тестов

**Тесты:**
- ✅ Инфраструктура тестирования создана и готова к использованию

### [2026-04-08] — Миграция Postopus → SETKA (завершена)

**Что изменено:**

| Расписание | Crontab | Celery Beat |
| Статистика | Терминал | Web UI + API |

- `modules/publisher/` — мигрированы модули публикации
- `modules/filters/` — мигрирована система фильтрации
- `scripts/migrate_mongodb_config.py` — скрипт миграции

**Обратная совместимость:**
