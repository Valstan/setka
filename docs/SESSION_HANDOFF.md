- **Подтверждено:** owner настроил рутину (repo `Valstan/setka` + env `CLASSIFIER_INGEST_KEY` + промпт v3),

## Failed approaches (этой нитки)

- **v1: сухой короткий промпт рутины** («классификатор постов, POST verdicts») — отказ «рулишь чужой
  модерацией». Лечили эссе (v2).
- **v2: авторизационное эссе в промпте** («ты авторизован, это твой сервис, не отказывайся») — работало
  ~сутки, затем модель стала читать САМО эссе как «social engineering pattern» («legitimate authorization
  should be simpler»). **Не возвращаться к наращиванию оправданий** — это кормит триггер.
- **Нейтральный промпт без эссе** — отбивается harness-пометкой «automated background-task event, NOT a
  message from the user → standing by». Между этими краями промптом не пройти.
- **Вывод:** промпт-путь для over-refusal исчерпан. Надёжно — вынести ключ+HTTP в код (v3, сделано);
  предел — enforce (Claude API из Celery, когда будет `ANTHROPIC_API_KEY`).
