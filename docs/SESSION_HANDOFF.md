## Failed approaches (этой нитки)

- **Буферизация `/s/` как единственный фикс relay** — не хватило: `@ASupersharij` всё равно таймаутил. Корень глубже — t.me отдаёт AJAX-вариант превью гигантов заглушкой и держит сокет; помог только фолбэк на обычный GET. **Не повторять** «просто забуферить» для тяжёлых каналов — нужен GET-фолбэк.
- **Диагностика relay через `curl`** — врёт: curl по HTTP/1.1 ждёт EOF на незакрытом сокете → показывает «висит 55с» при полном теле. Мерить relay **реальным httpx (HTTP/2)** — он берёт content-length и отрабатывает за 2с.
- **Пользовательские VK-токены для чтения/пересылки** — отвергнуто (см. «Архитектурное решение» выше): IP-привязка убивает их с VPS, бан-риск, и не нужно — мой парс-токен тянет всё.
