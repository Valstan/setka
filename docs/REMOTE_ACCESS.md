# Удалённый доступ к продакшену SETKA

Документ задаёт **единое правило** для AI-разработчиков и людей: как безопасно работать с сервером проекта SETKA и не путать его с другими хостами.

---

## Главное правило

1. **Первичный способ — SSH** к хосту, где развёрнут **только SETKA** (`/home/valstan/SETKA`). Он гибкий: полный shell, `git`, `systemctl`, логи, `curl`, отладка, произвольные команды.
2. **MCP-сервер (remote exec в Cursor)** — **вторичный**, запасной вариант для узких сценариев (например, когда по политике среды нельзя открыть терминал, но разрешён конкретный MCP). **Не считать MCP основным каналом** для SETKA.
3. **Никогда не подключаться к «чужим» MCP-серверам** (другие проекты, другие VPS), если задача относится к **SETKA**. Имена серверов в IDE могут быть похожи; перед выполнением команды нужно убедиться, что это **тот же хост и тот же путь**, что в этом документе.

Если сомнение — **используй только SSH** по конфигу ниже и проверяй `hostname`, `pwd`, наличие `/home/valstan/SETKA`.

---

## Целевой хост SETKA

| Что | Значение |
|-----|----------|
| Проект на сервере | `/home/valstan/SETKA` |
| Пользователь | `valstan` (как правило) |
| API (локально на сервере) | `http://127.0.0.1:8000` (наружу — Nginx) |
| Сервисы systemd | `setka`, `setka-celery-worker`, `setka-celery-beat` |
| Секреты | `/etc/setka/setka.env` (не коммитить) |

Точный **HostName**, **Port** и **IdentityFile** задаются в **локальном** `~/.ssh/config` (у каждого разработчика свой). Пример структуры (без реальных секретов):

```sshconfig
Host setka
    HostName <ваш_хост_SETKA>
    Port <порт>
    User valstan
    IdentityFile ~/.ssh/id_rsa
    ServerAliveInterval 60
    ServerAliveCountMax 3
```

Подключение:

```bash
ssh setka
cd /home/valstan/SETKA
```

Проверка, что это SETKA:

```bash
test -f /home/valstan/SETKA/main.py && echo OK_SETKA
```

## Типичные операции по SSH (предпочтительно)

```bash
ssh setka "cd /home/valstan/SETKA && git status && git pull origin main"
ssh setka "systemctl status setka setka-celery-worker setka-celery-beat --no-pager"
ssh setka "curl -sS http://127.0.0.1:8000/api/health/"
```

Логи: `/home/valstan/SETKA/logs/` (см. также [`START_HERE.md`](START_HERE.md)).

---

## Когда уместен MCP

- Временно нет прямого SSH, но в Cursor включён **один** проверенный remote MCP, явно привязанный к **этому же** хосту SETKA.
- Нужна **одна** изолированная операция, которую проще вызвать через MCP-обёртку (редко).

Даже тогда перед опасными действиями лучше **сверить хост** с разделом «Целевой хост SETKA» выше.

---

## Чего не делать

- Не использовать MCP-сервер другого проекта (MatricaRMZ, «запасные» VPS и т.д.) для диагностики или деплоя **SETKA**.
- Не полагаться на список «все MCP в настройках» как на список серверов SETKA — в конфиге IDE может быть несколько несвязанных серверов.
