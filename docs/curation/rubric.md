# Рубрика релевантности для LLM-курации дайджестов

> Растущая рубрика (а не дообучение модели) — носитель «практического обучения»
> фильтра. Ревьюишь ошибки → дописываешь правило/пример здесь → следующий прогон
> `/curate` умнее. Письмо brain 2026-06-07 «LLM-курация дайджестов» §«Обучение — честно».

Каждый кандидат в `digest_curation_runs.candidates` **уже прошёл** все
детерминированные фильтры (реклама/религия/дедуп/relevance) и был опубликован.
Задача LLM — поймать то, что алгоритм пропустил. Любой `drop` = чистая дельта
над текущим алгоритмом, поэтому **default = keep**: ставь `drop` только при
уверенности, иначе мы плодим ложные срабатывания (это и есть метрика precision,
которую мерим в Фазе 1).

## Вердикт `drop`, если пост попадает в один из классов

1. **Реклама / коммерция**, проскочившая ad-фильтр: продажа товара/услуги, набор
   на курсы, «пишите в директ», прайс, промокод, агрессивный призыв купить.
   *Не* реклама: анонс бесплатного районного мероприятия, вакансия от
   госучреждения, соцобъявление.
2. **Нерелевантно району**: событие/тема без привязки к данному району
   (общегородской/федеральный контент, омонимичный топоним другого региона,
   развлекательный репост без локального повода). Привязка = упомянут
   район/его населённый пункт/местное учреждение, **или** очевидно местное
   событие.
3. **Переписанный дубль**: смысловой повтор другого поста этого же прогона
   (перефраз, та же новость другими словами) — simhash такое не ловит. Оставляй
   первый/полнейший вариант (`keep`), помечай `drop` повторам с reason
   «перефраз-дубль #<lip первого>».

## Вердикт `keep` (оставить) — всё остальное

Нормальная районная новость, культурное/спортивное событие, объявление
госоргана, локальное происшествие, поздравление, объявление о пропаже и т.п.

## reason — кратко и машинно-группируемо

Короткая каноничная причина (она агрегируется в `--stats` → `drop_reasons`):
`реклама`, `не-район`, `перефраз-дубль`, `федеральный`, `развлекательный-репост`.
Можно дописать `: деталь`.

## Per-region заметки (дополняется по ходу PoC)

### <region_code> (название)

- _Накопленные особенности района: типичные омонимы, локальные паблики, частые
  ложные срабатывания. Дописывай сюда после ревью ошибок._

### mi (Малмыжский район)

- **Паблик `216215303`** постит ежедневные исторические заметки «N июня YYYY г. …»
  («в этот день в истории») — общероссийский патриотический контент без привязки
  к району. → `drop` `не-район: историческая дата без местной привязки`.
- **Областной (Кировская обл.) контент — основная серая зона.** Эвристика после
  PoC-прогона 2026-06-08:
  - `drop` (`не-район`): генеричные областные сводки/промо без местного хука —
    геомагнитные бури, всероссийские конкурсы, областные квест-туры с розыгрышем.
  - `keep`: областное с **практической пользой для жителя или призывом к действию** —
    предупреждение о непогоде с горячей линией энергетиков, голосование ФКГС
    (благоустройство, среди объектов есть местные), региональная прокуратура о
    мошенничестве. Граница: «могу ли я (житель Малмыжа) что-то с этим сделать
    прямо сейчас?» → да = keep.
- **Барахолка `24611937`**: продажи с ценой/телефоном — `реклама` (`drop`); но
  объявления о пропаже/находке животных и пристрой животных — соцобъявление
  (`keep`, рубрика §keep).
