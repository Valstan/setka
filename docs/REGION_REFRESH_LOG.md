# Журнал освежения регионов

> Учёт того, **когда и какой регион (район/область) последний раз освежался по канонам проекта**: добор свежих доноров (`communities`), чистка устаревших/мёртвых, перекатегоризация под текущую таксономию, применение новых фич платформы. Цель — чтобы ни один регион не отставал, когда проект развивается.
>
> Заполняется/правится **вручную в конце каждого освежения** (после прогона [`/discover_communities`](../.claude/commands/discover_communities.md)). Git-трекается — это и есть источник истины между машинами.

## Как пользоваться

1. Открыть таблицу [«Состояние регионов»](#состояние-регионов), отсортированную по приоритету освежения (бедные/давно не трогавшиеся — сверху).
2. Сказать ассистенту: «обновим следующий самый устаревший регион» → берём верхнюю строку с пометкой «🔴 не освежался» / самым старым «Освежён».
3. Прогнать освежение (см. [процедуру](#процедура-освежения)).
4. Обновить строку региона в таблице + добавить запись в [журнал событий](#журнал-событий).

## Канон-чеклист (что значит «освежён»)

Версия канонов на **2026-06**. При появлении новой фичи — добавить пункт сюда и пере-свипнуть регионы.

| # | Канон | Как проверить |
|---|---|---|
| C1 | **Gate-eligible** — регион попадает в тематические волны | есть активный пул `communities` (после [PR #108](https://github.com/Valstan/setka/pull/108) этого достаточно) ИЛИ `region_configs` ИЛИ `digest_mode='communities'` |
| C2 | **Брендинг** — строка `region_configs` с человекочитаемыми `zagolovki`/`heshteg`/`heshteg_local.raicentr` | `SELECT * FROM region_configs WHERE region_code=…` |
| C3 | **Локалитеты** — `regions.config.localities` (для discovery) + `region_configs.localities` (для RegionalRelevanceFilter) | непустой список сёл района |
| C4 | **Покрытие тем** — пул покрывает канон-темы района: `novost, reklama, kultura, sport, admin, union, detsad, sosed` | нет «тонких»/пустых тем (особенно novost/admin) |
| C5 | **Канон-таксономия** — категории по канону, без дрейфа (`detsad` ≠ школы; нет неканонной `other`) | см. таблицу тем в `/discover_communities` |
| C6 | **Свежесть пула** — мёртвые/спящие почищены, новые доноры добраны вручную скилом (нейро-в-петле) | `health_status`; ручной прогон `/discover_communities` |
| C7 | **Знак `vk_group_id`** нормализован (отрицательный) | [миграция 017](../database/migrations/017_normalize_region_vk_group_id_sign.sql) |

**Неканонные категории под чистку:** `other` (не входит ни в одну волну → группа простаивает) → перекатегоризировать в канон или отклонить.

## Процедура освежения

1. Снять срез региона: пул по категориям, `region_configs`, `config.localities`, `last_discovery_at`.
2. Почистить: неканонные категории (`other` и т.п.), мёртвые/нерелевантные, дрейф категоризации.
3. [`/discover_communities <code>`](../.claude/commands/discover_communities.md) — район-режим (`--localities`, `--main-group`, `--min-members 0`): скан → **классификация в чате по постам** → засев годного.
4. Обновить эту таблицу + журнал событий.
5. Доки/деплой: правки пула — DB (SQL/seed, без деплоя); правки кода/миграции — через [`/reliz`](../.claude/commands/reliz.md).

> ⚠️ Авто-discovery (`discovery-rolling-daily`) **отключён** с 2026-06-02 — без нейро-фильтра даёт ~98% шума (омонимы названий сёл, чужие регионы, коммерция). Освежение — только вручную скилом, пока к discovery не подключат нейро-классификацию.
