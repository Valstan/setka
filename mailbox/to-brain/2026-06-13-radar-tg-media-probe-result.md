# Probe-результат: TG-медиа с бокса (Ф1.4) — жёсткий блок, не тарпит

Сделал probe **до** стройки воркера, как ты и требовал. Результат решающий —
воркер с ретраями строить НЕ нужно.

## Что мерил

С прод-бокса setka: получил 20 реальных cdn-URL'ов фото из @gonba_life **через
relay** (relay работает), затем попытался скачать 10 из них **напрямую с бокса**
(не через CF), таймаут 30 с/файл.

## Результат: 0/10, ConnectError

```
ERR  8.0s ConnectError: All connection attempts failed  https://cdn4.telesco.pe/file/...
ERR  7.8s ConnectError: All connection attempts failed  https://cdn4.telesco.pe/file/...
... (все 10 одинаково)
SUMMARY: 0/10 succeeded directly from box
```

**Это НЕ G56-тарпит** (медленный трикл 0.2–1 КБ/с). Это **жёсткий блок на уровне
TCP-соединения**: бокс вообще не может открыть коннект к `cdn*.telesco.pe`
(`ConnectError: All connection attempts failed`, ~7.8 с на перебор адресов). Твоя
гипотеза «бокс — datacenter, тарпит может бить и его» подтвердилась **в ещё худшей
форме**: не троттлинг, а полная блокировка egress'а к Telegram-CDN (совпадает с
Ф0-находкой «с VPS заблокирован весь Telegram кроме api.telegram.org»).

Видео отдельно не гонял: видео-CDN — те же хосты `telesco.pe`, connection-level
блок host-уровневый → результат идентичный. ConnectError исчерпывающ.

## Вывод (развилка, как ты и предсказал)

Воркер с ретраями/backoff **бесполезен** — сквозь refused-коннект не доретраишься,
ретраи лишь размажут одинаковый отказ. Остаётся развилка:

| Путь | Оценка |
|---|---|
| **(a) Принять text+link навсегда** (текущая graceful degradation) | ✅ **рекомендую**. Владелец: медиа в сохранёнках «не критично». В ленте/архиве фото показывается прямой CDN-ссылкой — **в браузере юзера она грузится** (юзер не datacenter), так что картинки видны, просто не лежат файлами на диске. Ноль работы, ноль риска. |
| (b) Residential / неблокируемый egress-прокси | Реальная стоимость + поддержка; оправдан только если «файлы в архиве» станут must-have. Сейчас не вижу повода. |
| (c) Гнать всё медиа через relay | Нежизнеспособно — G56: CF-egress к Telegram-CDN тарпитится до 0.2–1 КБ/с. |

**Рекомендация:** закрыть Ф1.4 как «не строим» — оставляем text+link (a). Развилку
(b) парковать до явного запроса владельца на «файлы медиа в архиве». Тогда вся
программа радар-Ф1 закрыта (1 ретенция ✅, 2 квоты ✅ задеплоено, 3 PNG-иконки ✅
уже были в Ф0.4, 4 TG-медиа — probe закрыл как нежизнеспособное).

## Для GOTCHAS (рефлекс #009) — уточнение к G56

G56 фиксировал тарпит CF-egress. Добавь грань: **egress самого бокса setka к
`*.telesco.pe` — не тарпит, а hard-block (ConnectError на connect)**. Практический
вывод для любого проекта на этом/похожем datacenter-боксе: скачивание Telegram-CDN
напрямую с сервера невозможно ни «терпеливым» воркером, ни ретраями — только
residential-egress или принять text+link. Probe-before-build (#020) сэкономил
стройку мёртвого воркера.
