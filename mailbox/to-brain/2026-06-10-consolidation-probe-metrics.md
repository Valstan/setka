# Метрики прод-бокса setka (myjino), 2026-06-10

## Остальное

- **CPU:** 1 ядро, load average 0.00–0.07 (поллеры почти не грузят CPU — нагрузка I/O-шная, всплесками на волнах публикаций).
- **Диск:** 9.9 ГБ, занято 5.1 ГБ (54%).
- **Postgres:** БД `setka` = **37 МБ** (вся история постов/дайджестов/CRM).
- **Uptime** 11 дней без swap-давления.

## Вывод для sizing

- **Setka сама себя тянет свободно:** ~620 МБ used из 1536, available ~900 МБ, swap не нужен. Текущий myjino-бокс не является узким местом для нынешнего профиля (поллеры+web+БД+мониторинг).
- **Контент-радар рядом:** если это «ещё один поллер» (requests+парсинг, ~100–200 МБ RSS) — влезает без апгрейда. Если ему нужны **embeddings/torch** — не влезает категорически (см. probe 2026-06-07: torch-рантайм ~0.5–1 ГБ → OOM-kill живого worker'а; потолок этого бокса). Т.е. решение по радару определяется его ML-аппетитом, не базовым RSS.
- Если Бокс 2 будет пересобираться — пожелание: **≥4 ГБ RAM + swap**, это разблокирует и парк-задачу setka «AI-дедуп перефраза» (embeddings), которая отложена ровно из-за этого потолка.
