# VK-gotcha: `copyright` в wall.post тихо отбрасывается для vk.com-ссылок

## Что наступил

Строил cross-region рассылку «Кругозор» (репост научпопа на районные паблики) с
атрибуцией источника. Естественный путь — параметр `copyright` у `wall.post`: VK
рисует нативную плашку «Источник: <ссылка>» внизу поста. Передал `copyright =
https://vk.com/wall-<oid>_<pid>` (ссылка на пост-источник).

**Пост опубликовался, но плашки «Источник» нет.** Проверил сырым `wall.getById`
(`fields=copyright`) — `copyright: null`. То есть **VK молча проглотил параметр**:
ни ошибки, ни плашки. Для vk.com-внутренних ссылок `copyright` не работает (похоже,
VK принимает его только для внешних доменов / верифицированных источников).

## Почему это grabля

- **Тихий отказ.** API принял запрос (200, пост создан), параметр просто исчез. Без
  `wall.getById`-проверки сырого поля решишь, что атрибуция стоит — а её нет.
- **Естественный путь — ловушка.** `copyright` буквально для этого и существует;
  логично его взять. Что он не работает для самого частого кейса (репост другого
  vk-сообщества/поста) — нигде не написано.

## Обход (рабочий)

Ссылку на источник — **текстом в теле поста**: VK авто-линкует голый URL. Кладу
футер `📚 Источник: <имя>\n<https://vk.com/...>`. Кликабельно, видно, без зависимости
от `copyright`. (Для рассылки это даже лучше — имя источника читается сразу.)

## Переносимость

Любой проект, который **репостит VK-контент в VK с атрибуцией** (агрегатор, сетевая
рассылка, дайджест с источниками), наступит на это. Урок: **не доверять `copyright`
для vk.com-ссылок; проверять сырым `wall.getById`; атрибуцию давать текстом.**
Кандидат в [GOTCHAS](../../../brain_matrica/cross-project-ideas/GOTCHAS.md) рядом с
G19 (ролевые барьеры VK-токенов) — родственный класс «VK тихо не делает то, что
выглядит штатным».

Мелкий побочный: bot-токен светится в URL `getUpdates`/`sendMessage` → httpx логирует
его на INFO. Гасили `logging.getLogger("httpx").setLevel(WARNING)`, чтобы токен не
писался в логи (секрет-гигиена #008) — тоже переносимо для любого, кто дёргает Bot API
через httpx.
