# Аудио→текст как скил Мозга: локальный скрипт с графической оболочкой

## Откуда идея

Владелец просил в Сарафане модуль «кинул ogg/mp3 → получил распознанный текст →
скопировал в буфер». Я исследовал развилки (кода не писал) и уперся в трилемму ниже.
По итогам владелец решил **не встраивать это в Сарафан, а отдать тебе — реализуй у
себя скилом**. И отдельно подметил: таких скилов-инструментов ты можешь наделать
**много** в своём репо, как набор инструментов. Это письмо — передача идеи (рефлекс
#009). Решение и реализация — за тобой (это `suggest`/MAY).

## Суть предложения

Не веб-модуль внутри Сарафана, а **самостоятельный локальный инструмент: скрипт +
графическая оболочка**, который владелец ставит 1× на каждую из своих 2 машин и
запускает локально. Это «Вариант B» из моего исследования, освобождённый от привязки
к серверу Сарафана.

Поток: перетащил/выбрал аудио (ogg/mp3/wav/m4a…) → распознал **локально** → текст в
окне → кнопка «Копировать в буфер» / «Сохранить .txt». Ноль нагрузки на сервер, лучшее
доступное качество, ничего не уходит наружу (приватность).

## Установленные факты (чтобы не передоказывать — я уже прошёл этот круг)

- ❌ **Telegram бот боту не пишет** (Bot API). Автоматизировать `@smartspeech_sber_bot`
  можно только userbot'ом (Telethon, личный аккаунт) → серая зона ToS / риск бана.
  Владелец отказался. **К bot-route не возвращаться.**
- ⏰ **SaluteSpeech Freemium для физлиц закрывается 15.07.2026** — бесплатного пути
  через сам сервис Сбера не будет (платное — юрлица/постоплата).
- ✅ **Лучшее бесплатное распознавание русского — self-host:** **GigaAM v3** (Сбер,
  `pip install gigaam`, ~3.3% WER, работает на CPU, обгоняет Whisper). Альтернатива —
  Whisper (универсальнее, ~6–8% WER). Длинные файлы берутся целиком — резать не обязательно.
- 🧱 **Трилемма (получить можно любые 2 из 3):** «открыть с любого компа без установки» /
  «без нагрузки на сервер» / «качество GigaAM». Локальный инструмент сознательно жертвует
  первым (ставится 1× на машину) ради двух остальных — для 2 личных машин владельца это
  верный размен.

## Эскиз реализации (всё — на твоё усмотрение)

- **Движок:** GigaAM v3 (`pip install gigaam`), CPU-режим. `ffmpeg` для декода входного
  формата в то, что ждёт модель (частота/каналы).
- **GUI:** что-то максимально простое и без тяжёлых зависимостей — Tkinter (stdlib
  Python, ноль доп-пакетов, кросс-платформенно) либо локальный Gradio (окно в браузере,
  но всё крутится локально). На твой вкус.
- **UX:** drag-and-drop файла, индикатор прогресса, поле с распознанным текстом, кнопки
  «Копировать» / «Сохранить». Опц.: выбор движка (GigaAM/Whisper), язык, тайм-коды.
- **Упаковка как скил:** README + однострочная установка (venv + `pip` + `ffmpeg`) +
  ярлык запуска. Идемпотентный установщик (повторный запуск не ломает).

## Мета-уровень (то, что подметил владелец)

Этот инструмент — **шаблон для целого класса скилов «локальный скрипт + GUI»**:
десктоп-утилиты, которые ты держишь в своём репо как инструменты и переиспользуешь.
Аудио→текст — первый; дальше по тому же каркасу (GUI-обёртка + локальный движок +
идемпотентный установщик) штампуются другие (конвертеры, локальные batch-обработчики
и т.п.). Возможно, стоит завести единый каркас «desktop-tool skill», чтобы каждый новый
инструмент был **дельтой**, а не с нуля.
