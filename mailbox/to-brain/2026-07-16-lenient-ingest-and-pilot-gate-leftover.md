# Две находки: толерантный приём вывода агента + грабля «обкаточный гейт пережил обкатку»

## 1. Паттерн: вывод агента, стоивший токенов, принимай per-item и толерантно

**Боль (найдена в прод-логе):** ingest-эндпоинт SETKA принимал батч вердиктов
облачной рутины строгой pydantic-валидацией целиком. Один чинимый перелимит
(эхо текста поста длиннее лимита схемы, `confidence: 500`) → 422 на ВЕСЬ батч →
результат целого прогона рутины выброшен. Токены потрачены, вердикты потеряны,
часовой слот сгорел — и это **тихо**: рутина не ретраит, сервер логирует лишь
access-строку.

**Урок:** вывод LLM-агента — это не форма от фронтенда. Он (а) стоил реальных
денег, (б) статистически шумит по краям схемы (длины, диапазоны, типы).
Строгая whole-batch валидация превращает каждый шумовой хвост в потерю всего
прогона. Правильная граница:

- разбор **per-item**, батч не падает никогда (кроме совсем неструктурного тела);
- **чинимое чинить**: строки обрезать до лимитов схемы, числа зажимать в
  диапазон, мусорные типы приводить;
- **нечинимое считать**, не роняя остальных (`skipped_invalid` в ответе +
  warning в лог) — счётчик виден и агенту, и оператору.

Реализация — `parse_verdict_loose` в `modules/classifier/schema.py` (PR #354),
~40 строк. Применимо к любому соседу, принимающему структурированный вывод
LLM/рутины по HTTP (GONBA/Sabantuy-паблишеры, любые ingest-петли). Кандидат в
GOTCHAS: «422-на-весь-батч для agent-generated payload = тихая трата токенов».

## 2. Грабля: обкаточный гейт пережил обкатку и стал невидимой стеной

Фича катилась пилотом на одном регионе через два env-гейта: гейт самой фичи
(`CLASSIFIER_REGION_CODES`) и гейт её **источника данных**
(`COLLECTION_AUDIT_REGION_CODES`). При раскатке «на все регионы» переключили
первый, второй остался `=mi` — классификатор месяц «работал на всех регионах»,
видя только пилотный: enforce честно фильтровал, лента жила, ничего не падало.
Заметил владелец по косвенному признаку («мало постов на вердикт»).

**Урок:** у фичи, раскатываемой пилотом, гейтов обычно БОЛЬШЕ одного (фича +
источник + downstream). При promote пилота — грепнуть env/конфиг по имени
пилотного значения (`grep -r 'mi' /etc/…`), а не по имени фичи: остаточный
гейт источника не виден через гейт фичи. Дешёвая привычка, дорогая грабля.
