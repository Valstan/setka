# Радар-кабинет: инвентарь готового + probe доставки + дельта

По директиве `radar-personal-cabinet` (recommend, probe-first). Сделал ровно то,
что просила структура: сверился с Ф0/Ф1 (не переделывать), probe внешней доставки,
построил внутреннюю ленту-вывод + CRUD источников + (по зелёному probe) внешние выводы.

## 1. Что УЖЕ ЕСТЬ в Ф0/Ф1 (не дублировал)

| Возможность из директивы | Статус | Где |
|---|---|---|
| Точка входа в кабинет | **было** | radar-user логинится → лендинг `/radar`, изоляция AuthGateMiddleware (Ф0.1). Кнопку-лончер в опер-навигацию НЕ добавлял (канон изоляции: radar-user ≠ оператор). |
| «Раздел, куда новости сваливаются, кнопки под ними» | **было** | лента + архив Ф0.4 с per-item save/delete. Сделал её **типом вывода `feed`** (дефолт). |
| Источники «добавить ссылкой → список → вкл/выкл» | **частично было** | add/list/delete был; **довёл до CRUD**: добавил **паузу** (`is_active` на подписке, fan-out не страдает). Edit названия — не делал (источник резолвит title сам). |
| Delivery-adapters (наше прил./push/TG/VK) | **push был** | web-push Ф0.5. TG/VK-вывод — **построил** (см. §3). |
| Изоляция radar-user | **было** | вся дельта radar-user-scoped, чужое недостижимо (фильтр user_id). |

## 2. Probe внешней доставки (#020) — РАСХОЖДЕНИЕ С G63, важно для радара

**Ключевой факт: `api.telegram.org` ДОСТУПЕН с этого прод-бокса (myjino).**
- `curl https://api.telegram.org/` → **302 за 0.22с** (TCP-connect 0.046с), не таймаут;
- radar intake-бот `getUpdates` **успешно тикает каждую минуту** (лог worker);
- Telegram-алёрты (должники/watchdog/`mirror_community_to_telegram`) **уходят** — в коде уже
  есть исходящий постинг в TG-каналы (`modules/publisher/telegram_repost.py`).

→ **G63 на этом боксе НЕ материализуется.** Заблокированы только `t.me`/CDN (для них и нужен
tg-relay, read-only). **Bot API напрямую работает** — TG-вывод текстом relay НЕ требует.
Это per-box эмпирика: возможно стоит уточнить G63 гранью «бьёт t.me/CDN; api.telegram.org
зависит от провайдера РФ-бокса — проверять curl'ом, не предполагать» (у вас G63 говорит
«бьёт sendMessage И getUpdates» — на myjino это не так). Probe-before-build сэкономил
ненужную постройку relay-обёртки для Bot API.

**VK-вывод:** `wall.post` уже установлен probe-ами рассылки/обложек (16/16 can_post,
гибрид-токен снял G19) — переиспользовал `VKPublisher.create_with_policy` (как рассылка/krugozor).

## 3. Дельта построена (PR `feat/radar-cabinet-outputs`)

- Миграция 045: `radar_subscriptions.is_active` (пауза) + таблица `radar_outputs`.
- `modules/radar/delivery.py` — хук в поллере (после web-push, под kill-switch
  `RADAR_DELIVERY_DISABLED`): курсор **at-most-once** по `radar_items.id`, старт с MAX(id)
  (новый вывод не выстреливает бэклогом), bounded 10/прогон, throttle 1с, per-output изоляция,
  `fail_count`/`last_error` на выводе (видимость #018 — «вывод молчит/ошибка»).
- Типы вывода: `feed` (дефолт, no-op) / `telegram` (бот `sendMessage`, медиа в режиме «целиком») /
  `vk` (`wall.post`, текст+ссылка; **медиа в VK не рехостим** — атрибуция текстом, урок G64).
- Режим пересылки per-вывод: `excerpt_link` (начало+ссылка, дефолт) / `full` (целиком).
- API `/api/radar/outputs` CRUD + **`/test`** (кнопка «тест вывода» — взял из ваших suggest-добавок
  №1, «сильно рекомендую»: probe на уровне пользователя). UI: вкладка «Выводы» + пауза в «Источниках».
- +19 тестов (1400 зелёных), black/isort/flake8 чисто.

## 4. Из suggest-добавок — что взял и что в backlog

- **Взял:** «тест вывода» (№1). Индикатор здоровья источника/вывода (№4) — частично:
  `fail_count`/`last_error` уже показываются.
- **Backlog (по мере надобности):** instant vs дайджест per-вывод (№2), keyword-фильтр (№3),
  счётчик непрочитанного (№6). Дедуп меж источниками (№7) — ядро уже дедупит по `(source,external_id)`.

## 5. Открытый вопрос — мультиюзер-креды внешних выводов
