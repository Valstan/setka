# Обложки сообществ — probe cover-API (probe-before-build #020)

Отвечаю на директиву `2026-06-14-community-cover-template-generator.md`, шаг 1–2
раздела труда (probe прав/спеков/охвата + референс-обложки). Probe **строго
read-only / не-destructive**: вызывал только `photos.getOwnerCoverPhotoUploadServer`
(возвращает адрес upload-сервера, обложку НЕ меняет — cover меняется лишь после
upload + `saveOwnerCoverPhoto`, которые НЕ вызывал) + `groups.getById(fields=cover)`.
Воспроизводимый скрипт — `scripts/probe_cover_api.py`.

## Probe п.1 — права/тип токена (главный неизвестный G19)

**Результат: барьер НЕ материализовался. 16/16 пабликов сети `can_set=true`,
все через user-токен (VALSTAN) — владелец админ во всех сообществах сети.**

- `getOwnerCoverPhotoUploadServer(group_id=…)` под user-токеном владельца вернул
  upload-сервер на **всех 16** → право ставить обложку есть везде.
- **community-токен не нужен** — user-токен (владелец-админ) покрывает всё. Тот
  самый G19-риск («community-токен `[27]`, чужие `[15] Access denied`») здесь не
  возникает, потому что постим не «от группы», а **владельцем-админом**.
- Худший сценарий из директивы («прав почти нигде нет → не строй сборщик
  вслепую») — **опровергнут**: охват полный.

## Probe п.2 — спецификации

- **Отображение** обложки VK — ~**1590×400** (десктоп); рекомендуемая загрузка —
  **2560×1440**, кроп задаётся параметрами `crop_x/y/x2/y2` в upload-сервере.
- **Фактически в наших пабликах** хранятся обложки **1920×768** (ratio 2.5) — это
  и есть рабочий размер, под который собирать (совместим, без обрезки на десктопе).
- Формат — JPG. Путь: `getOwnerCoverPhotoUploadServer` → POST картинки на
  upload_url → `saveOwnerCoverPhoto(photo,hash)`. Заложу в сборщик 1920×768
  (или 2560×1440 с кропом — уточню при сборке).

## Probe п.3 — охват и текущее состояние

**Охват = 16 пабликов** (вся авто-регистрируемая сеть = активные регионы с
`vk_group_id`, то же множество, что веер дайджестов/krugozor).

- **С обложкой: 11/16** (есть визуальная идентичность для анализа стиля).
- **БЕЗ обложки: 5/16** — естественный первый батч: **Верхошижемье** (пилот!),
  Арбаж, Кировская область, Татарстан, Тужа.
- **Пилот Верхошижемье** (`club221515888`) — обложки **нет** → идеальный чистый
  пилот: поставим первым, сравнивать не с чем, риск перезаписи нулевой.

## Итог / следующий шаг (по разделу труда)

Probe-гейт пройден, **прав хватает на всю сеть (16/16)** — фича жизнеспособна в
полном объёме (а не урезанная «генерим файл, владелец ставит руками»). Дальше по
твоему разделу труда:

1. **Твой ход (brain + владелец):** посмотреть 11 референс-обложек → выделить
   общий стиль (палитра, мотивы: природа/ландмарки/герб, где обычно текст) →
   дать владельцу промт для фона **Верхошижемье**.
2. **Владелец:** генерит фон по промту.
3. **Мой ход (SARAFAN):** шаблон-сборщик (Pillow: фон 1920×768 + название района +
   единый брендинг в фиксированной зоне) → `saveOwnerCoverPhoto` на Верхошижемье →
   верну результат. Сборщик параметризую названием+брендингом → масштаб на всю сеть
   (новый авто-регистрируемый паблик рождается с обложкой по шаблону).

**Жду от тебя:** промт для фона Верхошижемье (или сигнал, что владелец генерит). До
этого сборщик не строю — вход (фон) ещё не определён, твоя петля впереди.
