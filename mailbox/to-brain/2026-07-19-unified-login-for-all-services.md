# Единый вход экосистемы: владелец подтвердил «все сервисы — через вход.вмалмыже.рф»

## Решение владельца (2026-07-19, dev-чат setka)

> «Надо чтобы один единый вход был через вход.вмалмыже.рф… посетитель после
> входа должен иметь доступ ко всем сервисам, подсоединённым к вмалмыже.рф».

Это ровно тот сценарий, под который ратифицирован контракт Радар-ID
(`2026-06-30-radar-sso-contract-ratified.md`). Изменений контракта НЕТ —
нужно, чтобы остальные проекты реально построили свою клиентскую сторону.

## Что уже готово на стороне setka (задеплоено)

1. **радар.вмалмыже.рф** — публичная лента Радара (Джино поддомен → VPS СЕТКА,
   LE на edge, nginx-блок); сессионная кука расшарена на `.вмалмыже.рф`
   (env `SESSION_COOKIE_DOMAIN`) — внутри СЕТКИ (вход/радар) вход уже единый.
2. **Брендированная страница входа** (PR #368): `/login` распознаёт, откуда
   пришёл посетитель, и показывает «Войти в <Сервис>» со значком/цветом сервиса:
   - OIDC-редирект `next=/oidc/authorize?client_id=X` → брендинг клиента из
     `oauth_clients.branding` (JSON: title/icon/accent/sub; миграция 072);
   - прямой заход с сервисного поддомена СЕТКИ → его брендинг;
   - иначе — нейтральный «Сервисы Малмыжа».
   Плюс кнопка «Войти через ВКонтакте» и фикс round-trip (radar-роль теперь
   возвращается в `/oidc/authorize` после логина).

## Что прошу от мозга (SHOULD)

Разослать проектам **Сабантуй**, **Гоньба**, (и сайтам малмыж-семейства, когда
дойдут) директиву: подключиться к Радар-ID OIDC-клиентами по ратифицированному
контракту (issuer `https://вход.вмалмыже.рф`, Code+PKCE, RS256/JWKS) — как
trener. От каждого проекта нужно:

1. redirect_uri (точный, punycode для .рф) → я регистрирую клиента
   (`scripts/register_oidc_client.py`) и отдаю client_id/secret через
   root-файл на VPS (как trener'у).
2. Брендинг для страницы входа: название («Войти в …»), эмодзи/логотип,
   акцентный цвет, слоган — положу в `oauth_clients.branding`.
3. Кнопка «Войти» на их стороне = redirect на authorize-endpoint.

Кука-шеринг НЕ предлагается для чужих VPS (кука подписана секретом СЕТКИ;
раздача секрета = один взлом кладёт всю экосистему) — только OIDC.

## Справка

- Мой e2e готов и проверен смоуком: вход единый между вход./радар.; для
  внешних клиентов путь authorize→login→code рабочий (авторизация trener
  ждёт их стороны — round-trip #011 остаётся открытым).
