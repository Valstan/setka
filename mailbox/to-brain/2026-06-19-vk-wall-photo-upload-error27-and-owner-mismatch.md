# Находка: заливка wall-фото VK — токен и владелец

Чинил баг setka: «Сетевая рассылка» публиковала **только текст**, прикреплённые
картинки молча терялись (посты выходили успешно, без ошибки в UI). Два урока,
оба расширяют уже зафиксированную в [GOTCHAS](../cross-project-ideas/GOTCHAS.md)
семью **VK error 27 «method is unavailable with group auth»** (там уже есть
`wall.edit` и `stats.get`).

## 1. `photos.getWallUploadServer`/`saveWallPhoto` под community-auth → [27]

Заливка фото на стену сообщества **community-токеном** падает с
`[27] Group authorization failed: method is unavailable with group auth` — ещё
один метод из семьи «роле-барьер #27». Грузить надо **user-токеном админа** группы.
Probe (read-only, `getWallUploadServer` лишь возвращает upload-URL, ничего не
меняет): **16/16** пабликов сети заливаются user-токеном, **0/16** community-токеном.

**Коварство:** баг **молчит**. `wall.post` фолбэчит на user-токен → текст уходит,
а пустой список вложений (заливка вернула None по каждой картинке) кэшируется →
все посты навсегда текстом. В UI/ответе API — успех. Видно только в логе worker'а.

## 2. Свежее wall-фото принадлежит группе → нельзя переиспользовать на чужой стене

Даже с правильным (user) токеном: `saveWallPhoto(group_id=X)` делает владельцем
фото **саму группу X**. Прикрепить такую attachment-строку (`photo-X_id`) к посту
**другой** группы Y нельзя — owner-mismatch, VK дропает вложение. Поэтому паттерн
«залить один раз, переиспользовать строку на все цели» для **свежезалитых** фото
структурно неверен — грузить надо **по каждой целевой группе отдельно**.

Контраст: репост/дайджест по **уже публичным** фото исходных постов
(`photoA_B`-объекты) переиспользуется свободно — публичный объект цепляется к любой
стене. Грабля только у **fresh upload**.

## Перенос

Любой проект, постящий медиа в несколько VK-сообществ от бота/сервиса, наступит на
оба. Кандидат в GOTCHAS как новые инстансы семьи error-27 (+ отдельная заметка про
owner-mismatch свежих wall-фото). Реализация в setka — `modules/broadcast/
dispatcher.py::_default_build_attachments` (user-токен + per-group), probe —
`scripts/probe_wall_upload_token.py`.
