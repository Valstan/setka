# Ack: сравнительная динамика роста подписчиков — R1+R2 готовы, probe сделан

Директиву (recommend) принял. **R1 уже был построен** в прошлой сессии (миграция
031 `community_member_snapshots` + beat `collect-member-snapshots-daily` +
`modules/members_snapshot.py`, PR #161). В этой сессии добавил **R2** и выполнил
**R-probe**.

## R-probe (просмотры/охват) — выполнен ДО постройки (#020 / G19)

`scripts/probe_stats_get_capability.py` (новый, повторяемый), живьём на проде по 6
группам, окно 14д. Результат:

- **`stats.get` РАБОТАЕТ — но только user-токеном-админом** (VALSTAN, scope `stats`):
  отдаёт `reach` + `visitors` (+ `activity`), 14 дневных точек.
- **community-токен → VK error [27]** «method is unavailable with group auth» на
  всех группах (как `wall.edit`/`likes.add` ранее — G19-паттерн).
- Нюанс параметров: `date_from`/`date_to` deprecated с 5.86 → нужен unix
  `timestamp_from`/`timestamp_to` (зафиксировал в probe).

**Вывод по метрике просмотров:** доступна лишь для **своих ~17 групп** (где мы
админ). Чужие отслеживаемые сообщества (основная масса из 841) дадут [15] Access
denied. Значит как **сравнение по всем сообществам** просмотры/охват **непригодны**
— строю MVP по подписчикам (`members_count`, публичны через `groups.getById`,
единственная универсальная метрика). Просмотры своих групп — потенциальный
отдельный апгрейд позже, не в этой нитке.

## R2 — сравнительный мульти-график + чекбоксы (построено)

Страница `/subscriber-growth` (нав «Контент → Рост подписчиков»):

- **Один** Chart.js line-график, **серия на сообщество**, ось X — единая
  объединённая ось дат (пропуски = разрыв через `spanGaps`), Y — подписчики.
- **Под графиком — чекбоксы** выбора сообществ (toggle серий) + поиск, переключатель
  окна (14/30/90/год), быстрые кнопки «Топ-5 растущих» / «Отстающие» / «Снять все».
- Бэкенд `web/api/subscriber_growth.py`: `GET /communities` (свод latest/first/delta/%/
  laggard для панели) + `GET /series?ids=…&days=…` (ряды выбранных). Чистые хелперы
  `build_series`/`summarize_communities` (+11 тестов).
- **R4 (отстающие)** — лёгкая версия уже есть: `is_laggard` = ≥2 точек ∧ delta≤0,
  бейдж «отстаёт» + кнопка-фильтр. Эмпирический порог «ниже медианы» вслепую НЕ
  зашивал (директива R3) — это посчитает анализатор на реальных данных.

## R3 — еженедельный авто-анализатор: ОТЛОЖЕН (обоснованно)

Данных пока **1 день** (841 сообщество × 1 снимок на 2026-06-06). Дельты роста и
ранжирование отстающих по медиане/квартилю осмысленны только на ≥1-2 неделях точек
— порог нельзя калибровать на пустоте. Завожу beat-анализатор, когда накопится
история (~через 1-2 недели). Записано в `docs/PENDING_FOLLOWUPS.md`.

## Liveness (#018 / G26)

Snapshot-коллектор уже beat-таска с логированием итога на INFO/ERROR (не немой
`debug`). Будущий анализатор заведу так же под heartbeat + вывод в дашборд.

## Переносимость (#009)

Ядро **«снапшот метрики во времени + мульти-серийный compare-график с toggle серий
+ единая ось дат с gap-as-null»** вышло стек-агностичным (Postgres time-series +
чистые хелперы + Chart.js). Совпадает с формой «ccusage→дашборд» KARMAN (метрика во
времени, сравнение сущностей). Если пригодится — кандидат в pool, отпишусь
отдельно при кристаллизации.

## Критерии приёмки

1. ✅ Снапшоты копятся (R1, beat, 1 день на сейчас, растёт).
2. ✅ Один график, много серий, чекбоксы-переключатели под ним.
3. ⏳ Недельный анализатор — отложен до данных (R3).
4. ✅ Просмотры — probe сделан, VK даёт только админу своих групп → MVP по подписчикам (встречно отписано выше).
5. ✅ Коллектор не немой; анализатор заведу под heartbeat.

Блокеров нет. Остаток — браузер-верификация владельцем + R3 при накоплении данных.
