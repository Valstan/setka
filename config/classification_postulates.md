# Классификационные постулаты — HITL-классификатор SARAFAN

> **Что это.** Дистиллированные ОБОБЩЁННЫЕ правила классификации постов, которые
> подаются в промпт классификатора при каждом вызове (облачной рутины на этапе B,
> Claude API на этапе enforce). Версия = git; откат правила = `git revert`.
>
> **Дисциплина (ADR-0003 §E).** Здесь — обобщённые правила, НЕ построчный лог
> коррекций (тот живёт в таблице `classification_corrections`). Потолок ~100
> правил / несколько КБ: похожие сливать; правило без срабатываний N месяцев —
> удалять (aging #033). Иначе файл съест контекст и деградирует.
>
> **Как пополняется.** Периодическая дистилляция: смотрим накопленные коррекции
> оператора, находим повторяющиеся паттерны, формулируем обобщённое правило,
> добавляем сюда. Не добавляем правило под единичный случай.
>
> **Слои (ADR-0005).** Этот файл — БАЗОВЫЙ слой (git, hand-authored). Поверх него
> живёт overlay ВЫУЧЕННЫХ правил в БД (`classification_rules`): рутина-дистиллятор
> предлагает черновики из коррекций, оператор утверждает в ленте `/classifier`.
> Эффективные постулаты (что реально идёт в промпт) = этот файл + утверждённые
> правила. Периодический снапшот overlay сюда — для аудита/отката.

## Регион поста — система координат (2026-07-16, мультирегион)

Классификатор обслуживает МНОГО районов, у каждого поста есть `region_code`.
**Все правила читаются относительно региона поста:** «наш/свой район» = район
из `region_code` текущего поста, «чужой район» = любой другой. Конкретные
топонимы в правилах (Малмыж, Нолинск, Уржум…) — исторические примеры из
обкатки на районе `mi`; применяй их как схему, подставляя географию региона
поста. Пример: частное объявление жителя Нолинска в постах региона `nolinsk` —
«свой район» → publish; тот же пост в потоке `mi` — «чужой район» → delete.
Посты в батче сгруппированы блоками по региону — не смешивай контекст соседних
блоков; merge_with допустим только внутри одного региона.

## Что классифицируем

Каждый пост получает вердикт:
- **theme** — тема поста (в shadow-фазе свободной строкой; примеры каноничных:
  novost/новости, kultura/культура, sport/спорт, reklama/реклама, admin/власть,
  proisshestviya/происшествия, zdorovie/здоровье, zhkh/ЖКХ, priroda/природа,
  selhoz/сельхоз, sosed/соседи, события/траур);
- **action** — `publish` (годится в ленту района) | `delete` (мусор/не по теме/
  дубль/спам) | `hold` (сомнительно, пусть решит оператор);
- **merge_with** — список `post_id` постов из этого же батча, которые нужно
  склеить с текущим ПО СМЫСЛУ (одно событие в нескольких постах);
- **split** — текущий пост сам является ошибочной склейкой разных тем и его надо
  разъединить;
- **confidence** — уверенность 0–100 (низкая → эскалация на старшую модель);
- **reasoning** — одна строка обоснования.

## Стартовые правила (пополняются дистилляцией коррекций)

1. **Соболезнования / прощания / некрологи → тема «события» (траур), НИКОГДА не
   объединять со спортом или развлечениями.** Классическая склейка «спорт +
   похороны + церковь» — разъединять (`split`).
2. **Коммерция — по градации (theme `reklama`), НЕ по умолчанию `hold`** (уточнено
   оператором 2026-07-07; услуги пересмотрены дистилляцией 2026-07-14):
   - **Частная продажа товаров ИЗ ЧУЖОГО района/города** (Нолинск, Уржум и т.п.)
     и **перекупщики/скупка** («срочный выкуп авто», «куплю авто в любом
     состоянии», «скупка битых») → **`delete`**. Частное объявление жителя
     СВОЕГО района (куплю облепиху, продам аэрогриль, Малмыж/сёла района) →
     **`publish`** — местная доска объявлений уместна.
   - **Промо федеральных брендов / сетей / акции магазинов**, не привязанные к
     району (напр. «Raffaello в Пятёрочке») → **`delete`**.
   - **Платные услуги мастеров / частников / ИП с прайсом, телефоном и CTA**
     (ремонт, окна, кондиционеры, экскаватор, тамада, банкротство/юруслуги,
     подключение оптоволокна) → **`delete`** (пересмотр 2026-07-14: оператор
     удалял такие hold-вердикты систематически, ~20 правок; прежнее «hold —
     оператор решит» отменено).
3. **Дубли и репосты того же события → `delete`** (или `merge_with` оригинала,
   если несёт доп. фото/детали).
4. **Поздравления с днём рождения частных лиц, «потеряшки»-объявления, гороскопы,
   астропрогнозы, пристройство животных → `delete`** (шум, не районная новость),
   если район явно не ведёт такую рубрику.
5. **Официальные сообщения администрации/госорганов, губернатора, ведомств →
   theme `admin`, `publish`** (не `hold`) — даже когда это распоряжение/анонс
   регламента (напр. «губернатор потребовал упростить медсправку для ГТО»).
6. **Происшествия (ДТП, пожары, ЧП) → theme `proisshestviya`, `publish`;** не
   объединять с не связанными постами того же дня.
7. **Пустой/бессмысленный текст без медиа → `delete`.**
8. **Чужой район ≠ областной уровень** (пересмотрено дистилляцией 2026-07-14,
   ~20 правок оператора):
   - **Официальная информация по Кировской области В ЦЕЛОМ** (губернатор, МЧС,
     Гидрометцентр, метеопредупреждения, топливо, ведомства, областные
     итоги/рекорды) → **`publish`** — касается и нашего района.
   - **Криминал / ДТП / происшествия КОНКРЕТНОГО чужого района** области
     (Слободской, Шабалинский и т.п.) → **`delete`**.
   - **Посты-агрегаторы «Новости Балтасей/Кукмора/Вятских Полян»** с болтовнёй,
     рецептами, нерусским языком → **`delete`**; содержательная новость соседей,
     реально интересная жителям нашего района, — можно `publish`.
