# Исправление ошибки -310 в Cursor

## Проблема
Ошибка `-310` при открытии сайта в Simple Browser Cursor обычно связана с проблемами SSL сертификата во встроенном браузере.

## Решения

### Решение 1: Использовать HTTP (временно)
HTTP версия сайта автоматически редиректит на HTTPS, но может помочь обойти проблему:

1. `Ctrl+Shift+P` → `Simple Browser: Show`
2. Введите: `http://3931b3fe50ab.vps.myjino.ru/`
3. Сайт откроется и автоматически переключится на HTTPS

### Решение 2: Использовать внешний браузер
1. `Ctrl+Shift+P` → `Simple Browser: Show`
2. Введите URL и нажмите `Ctrl+Click` на ссылке для открытия во внешнем браузере

### Решение 3: Открыть через Live Preview локального файла
1. Откройте файл `web/templates/index.html`
2. Правой кнопкой → `Show Preview`
3. Это откроет локальную версию без проблем с SSL

### Решение 4: Использовать Debug конфигурацию
1. Откройте панель Debug (`Ctrl+Shift+D`)
2. Выберите конфигурацию "Open SETKA Site"
3. Нажмите F5

### Решение 5: Проверить настройки Cursor
1. Откройте настройки Cursor (`Ctrl+,`)
2. Найдите `simpleBrowser`
3. Попробуйте отключить проверку SSL (если есть опция)

## Альтернатива: Открыть в системном браузере

Создайте скрипт для быстрого открытия:

```bash
#!/bin/bash
# Открыть сайт в системном браузере
xdg-open https://3931b3fe50ab.vps.myjino.ru/
```

Или используйте команду:
```bash
xdg-open https://3931b3fe50ab.vps.myjino.ru/
```

Сертификат валиден и работает:

- Действителен до: 2026-03-07
- Статус: OK

Проблема скорее всего в том, как Cursor обрабатывает SSL во встроенном браузере.
