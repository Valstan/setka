# Исправление редирект-петли

## Проблема
Браузер показывает ошибку: "Сервер перенаправил ваш запрос слишком много раз"

## Причина
Проблема может быть вызвана:
1. HSTS заголовок в браузере (кэш)
2. Конфликтующие конфигурации nginx
3. Проблемы с кэшем браузера

## Решение

### Шаг 1: Очистить кэш браузера
1. Откройте настройки браузера
2. Очистите кэш и cookies для домена `3931b3fe50ab.vps.myjino.ru`
3. Или используйте режим инкогнито

### Шаг 2: Очистить HSTS кэш
**Chrome/Edge:**
1. Откройте `chrome://net-internals/#hsts`
2. В разделе "Delete domain security policies" введите: `3931b3fe50ab.vps.myjino.ru`
3. Нажмите "Delete"

**Firefox:**
1. Откройте `about:config`
2. Найдите `security.tls.insecure_fallback_hosts`
3. Добавьте домен или очистите HSTS через настройки

### Шаг 3: Проверить конфигурацию
HSTS заголовок временно отключен в nginx для диагностики.

### Шаг 4: Использовать HTTP (временно)
Попробуйте открыть через HTTP:
```
http://3931b3fe50ab.vps.myjino.ru/
```

## Проверка
```bash
# Проверить редиректы
curl -I https://3931b3fe50ab.vps.myjino.ru/

# Проверить без следования редиректам
curl -v --max-redirs 0 https://3931b3fe50ab.vps.myjino.ru/
```

## Восстановление HSTS
После решения проблемы можно вернуть HSTS:
```nginx
add_header Strict-Transport-Security "max-age=31536000; includeSubDomains" always;
```
